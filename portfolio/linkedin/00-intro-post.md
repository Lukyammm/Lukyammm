## Technical version

A thread on building production web apps with an "unfashionable" stack — Google
Apps Script + Google Sheets — and making them behave like real software.

What I've shipped (all internal hospital tools):
• Access-control app (lockers, visitors, audit) with caching + document locks +
  retry/backoff for safe concurrent writes.
• Process/KPI system with scheduled snapshot jobs, schema migrations, and
  server-enforced role-based permissions.
• Rapid-response intake with salted+hashed passwords, timing-safe compare,
  brute-force lockout, and SLA tracking.
• Satisfaction analytics with sector canonicalization (fuzzy matching),
  dedupe, and server-computed NPS.

The lesson: the platform doesn't make the software serious — the engineering
does. Sheets has quotas and no SQL, so you cache, lock, validate, and migrate
deliberately.
