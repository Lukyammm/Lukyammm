# Portfolio Package — Lukyam Madeira

This folder is a complete, ready-to-use kit for turning the GitHub account into a
recruiter-ready portfolio and launching it on LinkedIn. It was generated from a
**code-level audit** of every repository — not from the existing READMEs — so the
claims here match what the projects actually do.

## What's inside

| File | Purpose |
|---|---|
| `01-PROJECT-AUDIT.md` | Full audit of all 22 repos, impact classification (High/Medium/Low), and the detailed audit table. **Start here.** |
| `02-GITHUB-SETUP.md` | Repo descriptions, topics/tags, which 6 repos to pin, and clean-up actions. |
| `03-SCREENSHOT-GUIDE.md` | Exact, shot-by-shot screenshot plan for each featured app (these are Apps Script Web Apps — screenshots must be captured from your live deployment). |
| `readmes/` | Ready-to-paste, recruiter-facing **English** READMEs for the 8 featured projects. |
| `linkedin/` | LinkedIn posts for the top projects + an intro post, each in 4 variations (short / medium / storytelling / technical). |
| `04-PUBLISHING-PLAN.md` | Publishing order, slide captions, CTAs, and the LinkedIn approval gate. |

## How to use it (quick path)

1. **Read `01-PROJECT-AUDIT.md`** — agree on which projects to feature.
2. **Copy the READMEs** from `readmes/` into each repo (replace the generic ones).
3. **Capture screenshots** following `03-SCREENSHOT-GUIDE.md` into each repo's
   `docs/assets/`.
4. **Apply `02-GITHUB-SETUP.md`** — descriptions, topics, pinned repos, clean-up.
5. **Follow `04-PUBLISHING-PLAN.md`** for LinkedIn — with your approval at each step.

## Two honest constraints (read this)

- **Other repos couldn't be edited automatically from here.** This session's write
  access is limited to the profile repo (`Lukyammm/Lukyammm`), so the improved
  READMEs are delivered here as files for you to paste into each project repo.
- **Screenshots aren't pre-generated.** These apps only render with their real
  Google Sheets backend and an authenticated session, so genuine screenshots have
  to come from your live deployments. Faking them would misrepresent the work —
  instead, `03-SCREENSHOT-GUIDE.md` tells you exactly what to capture.
