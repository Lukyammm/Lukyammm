## About Me

My portfolio is built around real operational problems: monitoring indicators, organizing follow-ups, improving reporting routines, reducing spreadsheet friction, and making hospital workflows easier to track.

My strongest fit is not only writing code or queries. It is understanding an operational process, finding the data behind it, building a clear reporting structure, and delivering something that a non-technical team can actually use.

## Technical Strengths

- **BI and reporting:** KPI design, dashboard structure, filters, summaries, export-ready views, and decision-support reporting.
- **Healthcare operations context:** patient safety indicators, clinical process tracking, response workflows, reporting cadence, and privacy-aware documentation.
- **Workflow automation:** Google Apps Script, Google Sheets workflows, CSV/XLSX intake, validation steps, notifications, and internal tools.
- **Data quality:** checks for missing values, duplicates, inconsistent dates, invalid categories, and unreliable operational records.
- **SQL as support:** relational structure, joins, aggregations, KPI queries, and validation queries when a reporting workflow needs a stronger data layer.
- **Frontend for internal tools:** JavaScript, HTML, CSS, responsive dashboards, filters, modals, and practical user interfaces.
